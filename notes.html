Git Notes

  • The Git Flow = create repo, init git, stages changes, commit changes, push changes, pull request, merge to main branch.
  • Git init on terminal to create a new repos
  • "git add ."" to stage all changes at once
  • check if commit messages are past or present tense. be consitent and always use one
  • keep commits atomic, easier to rollback
  • "git commit --amend" = to add files that were missed out
  • prepare a .gitignore file to ignore files for committing. But .gitignore file must be committed. Also check gitignore.io
  • Branching is for = A/B Testing or debugging code. Or two person working on same thing. Or someone is working on a overhaul of the project.
  • HEAD is a branch pointer, pointing to which branch are you in
  • Daily use = git add, git commit, git branch, git status, git reflog
  • git config --local "or --global" user.name 'Lewis Lim' = adds your name in
  • git config --local "or --global" user.email 'email address' = adds your email address in
  • git log --oneline = summarize and displays logs in one line each. very neat
  • git branch "branch name" to create new branch on CURRENT BRANCH
  • git switch "branch name" to switch branch
  • git branch -d "branch name" to delete branch
  • git branch -m "branch name" to rename branch
  • git merge "branch name" to merge branch to current branch
  • git diff compares staging area and working directory
  • git diff HEAD compares HEAD's commit vs current working directory
  • git diff --staged shows what will change if you commit now
  • git diff --staged 'filename' = same as above but only for 1 file
  • git diff branch1..branch2 to compare 2 branches differences
  • git diff commit1hash..commit2hash to compare 2 commit differences. use git log to get hashes
  • git stash will save changes in staged and unstaged in current branch. can stash multiple times
  • git stash list to view list of stashes. use the git stash apply stash@{2} to choose a stash to apply
  • git stash pop to resume your stashed work and remove the stashed work
  • git stash apply to apply stashed work to multiple branches
  • git stash drop stash@{2} to choose a stash to drop
  • git checkout "id" moves you to a previous commit. do git switch "branch" to go back. git checkout HEAD to discard changes at old commit and go to HEAD.
  • git checkout origin/main = go back to the point where the github is
  • git restore 'filename' = restore file to its state in the HEAD
  • git restore --staged 'filename' = unstage files
  • git reset 'commit hash id' to pass a old commit to working directory, the new commits are gone.
  • git reset --hard 'commit hash id' same as above but remove from working directory also
  • git revert 'commit hash id' creates a new commit with reversed changes. safer than git reset.
  • git remote add origin 'https://github.com/ChumoroCat/Finding-Kitty' or see repo page, to point git to github
  • git remote -v to see which remote poing to
  • git clone 'github repo link' to download the repo and commits to my desktop
  • git push -u origin "main or branch name" to push from git to github
  • git push does not push tags. To push tags, use "git push origin 'tag name'"
  • git branch -r to view remote branches of github
  • git switch "branch name" to create a local branch of the same name as the remote branch, and connects it
  • git fetch origin master = updates your local branches based on the remote(github) branches for master
  • git pull origin main = git fetch + git merge
  • Pull request is a function on github. To seek approval for merging the branch made, to the main branch.
  • git branch origin = for fixing merge requests, to fetch everything. Then do git switch "branch name". Then merge branch to get the conflict to resolve. Then switch to main and merge again.
  • semantic versioning (1.0.1) = Ones are bug fixes, Tens are minor releases, Hundreds are major releases
  • git tag = prints a list of all the tags
  • git tag -a "v1.5.0" = annotated tags (can write description about the tag)
  • git show "v1.5.0" = show more information about the tag (tagger name etc)
  • git tag 'tag name' 'hash id' = to tag a previous commit
  • git tag -d 'tag name' = delete tags
  • git reflog = shows a log of what you did on git
